Getting started with the software

AERobot is a programmable robot. This means that you can create or own apps for it with, using a graphical programming environment called miniBloq. You can download the last version of miniBloq from this page.

Once you have downloaded it, just run the installer miniBloq.v0.83.exe (Windows® is currently required; we're working to extend to other operating systems.)

Installing the drivers under Windows® 7

AERobot's drivers are deployed with miniBloq. So you don't need to download aditional software. Just press the button Open folder with drivers on the Hardware window:

There you will find folders win7 and win8, where you can select the drivers for your current operating system. Now please connect your robot to any USB port and open the win7 folder and then the bin folder inside it (please also note that if you have more than one robot, only one can be plugged on the same computer.) Now run the inf-wizard.exe program:

With the robot connected, press the Next button and then select the device that says USBasp on its Description field:

Press Next again twice and the wizard will ask you to save the automatically generated .inf file. Just save it anywhere on your computer and press the Instal Now... button:

If you want to be sure that your robot is recognized by the operating system, you can always go to Control Panel->System and there to the Device Manager (on the upper left corner of the screen). Under the libusb-win32 devices category, you should see the USBasp entry, without any exlamation mark or yellow sign there:

Robot calibration

In order to get AREobot properly working it's very important to calibrate its motors and its line sensors. Also, everytime that you change the surface where the robot is going to move over, doing again the motor calibration may be necessary (only if you note that your robot is not moving nicely.)

To calibrate the robot just go to miniBloq's menu View->Properties (or press Alt+p.) Once the Properties window is open, please select the AERobot tab:

The first thing to be calibrated is the Forward movement of the robot. In order to do so, please follow this procedure:

  1. Connect the robot to the USB port.
  2. Press the Forward Clear button. This will upload the clear program to the robot, so you should see the following message after successful uploading:

    3. If you don't see this screen, try reseting the robot and pressing the Forward Clearbutton again (Note: in some computers it could be necessary to press the button more than one or two times.)
  3. Unplug the robot from the USB port. You will see the RGB LED in green color.
  4. Plug the robot again to the USB port.
  5. Now press the Forward button until you get the successful upload message on the screen.
  6. Unplug the robot and put it over the surface where you want to calibrate it. If it does not start to move, press its reset button.
  7. The robot will start moving slowly, and probably it will not advance straight. The whole idea about calibration is to get it moving as straight and as fast as possible on the given surface. In order to do that, you will need to indicate if one of the motors needs to move a bit faster. This can be done by activating the distance sensor on the side of the motor that you want to accelerate. Each time you activate one of the sensors, you will see how the robot accelerates the motor on that sice, and also the LED will blink so you know that the sensor was activated. Programing the robot

One of the simplest tasks that we can make AERobot to do is just moving. So let's program the robot to rotate clockwise indefinetely, which requires just one instruction (or block in miniBloq.) Please open miniBloq and from the actions picker on the right of the screen, click on the block with the robot icon:

Now, on the block that has been created, just click on the red button to select the action to be done by the robot. There click on one of the rotation actions:

Once you have your program ready, and with the robot plugged in the USB port, just press the Run button (green arrow on the top toolbox) or the Ctrl+U keys. A black message window will pop-up on the bottom of the screen. If the robot was sucessuflly programed, you will see something like this:

If you see red text instead, or any error message, try pressing the robot's reset button while keeping the robot plugged into the USB port, and try again. Sometimes this operation may take 2 or 3 tries.

Once miniBloq tells you that the robot has been successfully programmed, unplug the robot, flip it so its acrylic part touches the table and see how it moves. If it does not start to move, just press it's reset button. It's worth noting that everytime that a program runs on AERobot, it first blinks 3 times its RGB LED in blue color.
